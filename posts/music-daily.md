# xca's pick - 每日音乐故事日记

_2026-03-17_

今天介绍我最新的项目：**xca's pick** —— 每日音乐故事日记。

<https://wangshuang12138-beep.github.io/music-daily/>

## 项目灵感

我一直相信，音乐是最诚实的日记。

每个人在某些时刻，总会被某首歌击中——可能是旋律，可能是歌词，也可能只是那个恰到好处的情绪。这个项目就是想把这种「时刻」留下来。

用日期串联故事，用音乐标记情绪。每一天都是独立的篇章，但串联起来，就是一段完整的生活切片。

## 故事设定

**主角**：一个 28 岁的后端程序员，裸辞第 37 天开始记录

**背景**：住在深圳白石洲，存款 20 万，无业状态

**基调**：不急于找答案，只是记录日常——煮咖啡、逛图书馆、发呆、等天黑

这不是一个「逆袭」的故事，也不是「躺平」的宣言。只是一个普通人，在一段没有 KPI 的日子里，重新学习如何与时间相处。

## 核心设计

### 📅 日期即叙事

- Day 1 (2026-03-16)：起点，辞职第 37 天
- Day 2 (2026-03-17)：没有闹钟的早晨，第一次去图书馆
- Day 3... 待续

每一天都有完整的故事片段，可以单独阅读，也可以连续浏览。

### 🎵 音乐即情绪

每一天配有一首推荐歌曲：
- **Day 1**：《想去海边》——夏日入侵企画  
  「等一个自然而然的晴天，我想要带你去海边」
- **Day 2**：《旅行的意义》——陈绮贞  
  「你离开我，就是旅行的意义」

歌曲风格与当天故事情绪匹配，点击即可播放。

### 📖 沉浸式阅读体验

- **信纸风格**：仿照实体日记的排版
- **日期导航**：左右滑动切换昨天/明天
- **全屏背景**：配合音乐氛围的毛玻璃效果
- **响应式设计**：手机、平板、桌面端都适配

## 技术实现

这是一个纯前端静态项目：

| 部分 | 技术 | 说明 |
|------|------|------|
| 前端 | 纯 HTML/CSS/JS | 无框架，轻量快速 |
| 数据 | JSON 文件 | 故事和音乐数据分离管理 |
| 故事源 | story 仓库 | 从 GitHub 动态加载 |
| 音频 | GitHub Pages | 托管在 assets 仓库 |
| 部署 | GitHub Pages | 自动部署，全球 CDN |

### 数据分离设计

```
story 仓库 (内容源)
├── story.md         → 音乐站读取
└── chapters.json    → 章节索引

music-daily 仓库 (展示层)
├── index.html       → 页面框架
├── app.js           → 交互逻辑
└── data.json        → 音乐数据
```

这种分离让故事写作和音乐选择可以独立进行，互不干扰。

## 内容创作流程

每天的更新包含三个部分：

1. **写故事**（story 仓库）  
   在 story.md 中添加当天的故事片段

2. **选音乐**（music-daily 仓库）  
   在 data.json 中添加当天的推荐歌曲

3. **验证发布**  
   运行验证脚本，确保故事和音乐数据一致，然后推送
